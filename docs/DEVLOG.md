# Onjeom 개발·실수 로그 (내부용)

GitHub 커밋 메시지와 별개로, **무엇을 잘못했는지 / 무엇이 깨졌는지 / 어떻게 고칠지**를 남긴다.  
다른 AI·사람·미래 자신이 같은 함정을 반복하지 않기 위함.

**앱 버전 기준:** v0.4.9  
**로그 작성일:** 2026-07-14

---

## 규칙 (이번 사건 이후 강제)

1. **제품에 이미 있는 UI/기능을 쓰지 않고 API·우회를 새로 넣지 말 것.**  
   예: 테마 전환은 **설정 → 읽기 테마 칩**이 이미 있음. E2E 전용 `setReadingTheme` 추가 금지.
2. **요청 해석을 확인한 뒤 작업.** “스크린샷” ≠ “모든 포맷 캡처”. 디자인·취향(색 테마)이 목적이면 테마별 컷.
3. **되던 것을 건드리지 말 것.** 캡처 스크립트·문서 링크만으로 해결되면 `App.tsx` / 빌드 산출물을 건드리지 않음.
4. **무한 루프·불확실한 정리 로직 금지.** 실패 시 타임아웃 후 종료, 로그 남김.
5. **문서 이미지 경로와 실제 파일이 항상 일치**해야 함. 파일 삭제·이름 변경 시 README/docs 동시 수정.
6. 작업 끝날 때마다 이 파일에 **오류 / 증상 / 원인 / 수정 방안 / 상태**를 한 줄이라도 추가.
